- HTTP/2.0知道么？
    > HTTP2.0的多路复用和HTTP1.X中的长连接复用有什么区别？
        HTTP/1.* 一次请求-响应，建立一个连接，用完关闭；每一个请求都要建立一个连接；
        HTTP/1.1 Pipeling解决方式为，若干个请求排队串行化单线程处理，后面的请求等待前面请求的返回才能获得执行机会，一旦有某请求超时等，后续请求只能被阻塞，毫无办法，也就是人们常说的线头阻塞；
        HTTP/2多个请求可同时在一个连接上并行执行。某个请求任务耗时严重，不会影响到其它连接的正常执行.

- 用过什么代码仓库，git拉代码用的是什么命令，git上传的时候发生冲突如何解决，git的底层原理知道么？
    - CVS及SVN都是集中式的版本控制系统，而Git是分布式版本控制系统。
    - 集中式版本控制系统最大的毛病就是必须联网才能工作，完整的版本库只存在于“中央服务器”。
    - 分布式版本控制系统没有“中央服务器”，每个人的电脑上都是一个完整的版本库，这样，你工作的时候，就不需要联网了，因为版本库就在你自己的电脑上。
    - 安全性：分布式版本控制系统的安全性要高很多，因为每个人电脑里都有完整的版本库，某一个人的电脑坏掉了不要紧，随便从其他人那里复制一个就可以了。而集中式版本控制系统的中央服务器要是出了问题，所有人都没法干活了。
    - 当Git无法自动合并分支时，就必须首先解决冲突。解决冲突后，再提交，合并完成。解决冲突就是把Git合并失败的文件手动编辑为我们希望的内容，再提交。用git log --graph命令可以看到分支合并图。
- git是一个开源的分布式版本控制系统，分布式系统和单台的服务器有什么区别？只用单台的服务器会遇到什么问题？
    - 分布式系统和单台的服务器有什么区别？吞吐量？单台服务器根本无法承载大量访问请求和调用。
- 还了解过什么代码仓库吗？
- 对云是怎么理解的？有一台远程的服务器可以访问，和云产品有什么区别？
    - 云服务器和普通服务器主要区别有三点：

        1.定义不同：云服务器，是简单高效、安全可靠、处理能力可弹性伸缩的计算服务，是一个服务器集群；普通服务器是一个服务器，位置相对固定，是提供计算服务的硬件设备。
        2、配置不同：云服务器无需提前购买硬件，即可迅速创建或释放任意多台云服务器，一切计算均在云端实现，降低开发运维的难度和整体IT成本；普通服务器的构成包括处理器、硬盘、内存、系统总线等，和通用的计算机架构类似，费用成本较高。
        3、故障率不同：云服务器是基于服务器集群的，因此硬件冗余度较高，故障率低；而物理机则相对来说硬件冗余较少，故障率较高。

主键和索引的区别
1.主键一定是唯一性索引，唯一性索引并不一定就是主键
2.一个表中可以有多个唯一性索引，但只能有一个主键
3.主键列不允许空值，而唯一性索引列允许空值
